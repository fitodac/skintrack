# Setup

## Install dependencies

```bash
npm install
```

## Create local environment variables

```bash
cp .env.local.example .env.local
```

Set these values before starting local Supabase:

```bash
NEXT_PUBLIC_APP_URL=http://localhost:3000
SUPABASE_AUTH_EXTERNAL_GOOGLE_CLIENT_ID=your-google-client-id.apps.googleusercontent.com
SUPABASE_AUTH_EXTERNAL_GOOGLE_SECRET=your-google-client-secret
SUPABASE_PRODUCTION_DB_URL=postgresql://postgres.your-project-ref:your-db-password@db.your-project-ref.supabase.co:5432/postgres
```

## Start local Supabase in Docker

```bash
npm run db:start
```

## Write local app credentials into `.env.local`

```bash
npm run db:env:local
```

This command updates only the Supabase keys used by the app:

```bash
NEXT_PUBLIC_APP_URL
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY
SUPABASE_SERVICE_ROLE_KEY
```

## Regenerate database types

```bash
npm run db:types
```

## Local schema files

The local Supabase project is configured in:

```txt
supabase/config.toml
supabase/migrations/20260312180000_init_skintrack.sql
```

The local config keeps auth aligned with production behavior:

- public signup disabled
- email/password available only for invited users
- Google OAuth enabled through environment variables
- local callback allowed at `http://localhost:3000/auth/callback`

## Sync production data into local Supabase

To fully replace local data with production data:

```bash
npm run db:sync:prod-to-local
```

This command:

1. starts local Supabase if it is not running
2. resets the local database from migrations
3. dumps all `public` tables from production
4. also copies `auth.users` and `auth.identities`
5. restores the dump into local Supabase

Requirements:

- `SUPABASE_PRODUCTION_DB_URL` must be present
- `pg_dump` and `psql` must be installed locally

The sync intentionally does not copy active sessions or refresh tokens.

```bash
npm run dev

npm run db:reset
npm run db:stop
```

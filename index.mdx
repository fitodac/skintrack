# SkinTrack

SkinTrack is a private operational app for cosmetology practices.

It ships with:

- secure staff authentication with Supabase Auth
- patient records with ownership-based access
- clinical sessions with draft mode and autosave
- superadmin-controlled invitations and user management
- Docker-based local development with Supabase CLI
- production-to-local database sync for realistic local testing

## Stack

- Next.js 15 App Router
- Supabase Auth + PostgreSQL + RLS
- Supabase CLI for local services
- Docker Desktop for the app container and local Supabase
- Tailwind CSS 4
- Motion
- Zustand
- Zod
- Vitest

## Current modules

- Auth
- Profile
- Users
- Patients
- Clinical Sessions

## Local development model

The current `supabase-local` branch is built around a local Docker stack instead of a host-only setup.

- `npm run local:up` is the recommended entrypoint
- Supabase local runs in Docker through the CLI
- the Next.js app runs in the `skintrack-app` container
- `.env.local` is partially generated from `supabase status -o env`
- `SUPABASE_SERVER_URL` lets the app container reach the Supabase API on the host bridge
